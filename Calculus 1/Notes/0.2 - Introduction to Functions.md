## 1. What is a Function?

### Core Definition

A **function** is a rule (or relationship) such that every input produces **exactly one** specific output.

```
Function:  each input  →  exactly one output
```

> This is the non-negotiable requirement. If a single input can yield two or more different outputs, the relation is **not** a function — you cannot know where to go.

### Key Distinction: "one output" vs "unique output"

- **Function:** every input → exactly one (specific) output. Multiple inputs can share the same output — that is allowed.
- **One-to-one function:** every input → one unique output, AND no two inputs share the same output. (Tested with the horizontal line test — covered in Section 5.)

### Fish Table Example (Intuition)

| Fish (Input) | Weight in lbs (Output) |
|:---:|:---:|
| Fish 1 | 3.2 |
| Fish 2 | 1.4 |
| Fish 3 | 2.8 |
| Fish 4 | 7.3 |

This **is** a function — each fish has one specific weight. The scale is the "function": input a fish, get out one weight.

If Fish 1 weighed both 3.2 lbs and 4.7 lbs — that would **not** be a function. One input, two outputs = undefined behaviour.

### What if two inputs share an output?

```
Fish 1 → 3.2 lbs
Fish 4 → 3.2 lbs     ← same output, different inputs — STILL a function
```

This is allowed. It just means the function is **not** one-to-one.

### Mapping Diagram (Visual)

```
   INPUTS (X)          OUTPUTS (Y)
  ┌──────────┐         ┌──────────┐
  │   x₁    │────────▶│   y₁    │
  │   x₂    │────────▶│   y₂    │
  │   x₃    │────────▶│   y₁    │  ← same output: fine
  └──────────┘         └──────────┘
         ✅ Function (each input → exactly one output)

  ┌──────────┐         ┌──────────┐
  │   x₁    │──┬─────▶│   y₁    │
  │         │  └─────▶│   y₂    │  ← one input → two outputs!
  └──────────┘         └──────────┘
         ❌ Not a function
```

### Formula Example

Area of a circle:

```
A = πr²
```

This is a function because for any given radius r, there is exactly one area. No radius gives you two different areas.

## 2. Function Notation: f(x)

### Why not just use y = ?

When we write `y = 3x² - 4x + 2` and have multiple functions at once, "y" becomes ambiguous — which y are you talking about? Function notation solves this by labelling each function:

```
f(x) = 3x² - 4x + 2
g(x) = x + 1
h(x) = √(x - 3)
```

Each has its own identity. You can refer to f, g, or h specifically.

### What f(x) communicates

`f(x)` means: "the output of function f when x is the input."

It also creates a coordinate:

```
f(0) = 3(0)² - 4(0) + 2 = 2
```

This tells you: input was 0, output is 2 → coordinate point **(0, 2)**.

With `y = ...`, you know the output but not what went in. With `f(x)`, you know both.

### Evaluating f(x) from a table

Given the table:
| x | f(x) |
|:---:|:---:|
| 0 | 2 |
| 1 | 5 |
| 2 | 4 |
| 3 | 6 |

- f(0) = 2   → go to input 0, read the output
- f(3) = 6   → go to input 3, read the output

## 3. Representations of Functions

Functions can be described four ways:

| Representation | Example |
|---------------|---------|
| **Table** | Fish caught vs. weight |
| **Graph** | A parabola on an xy-plane |
| **Formula** | f(x) = x³ - 2x + 1 |
| **Mapping diagram** | Arrows from inputs to outputs |

Most of Calculus works with **formulas and graphs**. Understanding both — and the connection between them — is essential.

## 4. The Vertical Line Test

### Concept

A graph represents a function if and only if every vertical line drawn on the graph **touches the curve at most once**.

If any vertical line touches the graph at **two or more points** → it is **not** a function (one input, two outputs).

### Visual Walkthrough

**Case 1: Diagonal line (y = x)**
```
   y
   |  /
   | /
   |/
───+────→ x
   |
```
Every vertical line hits the diagonal at exactly one point. **✅ Function.**

---

**Case 2: Parabola (y = x²)**
```
   y
   |
   |  ∪
   | / \
   |/   \
───+─────→ x
```
Every vertical line hits the parabola at most once (the vertex is only touched once; either side is hit once). **✅ Function.** (Not one-to-one, but still a function.)

---

**Case 3: 1/x (reciprocal function)**
```
   y
   |   /
   |  /
   | /
   |
   |       (gap at x = 0)
   |
   |         \
   |          \
   |           \
───+────────────→ x
```
x = 0 is undefined — **no output at x = 0**. That is fine. A function does not need an output for every possible input, but if it does have an output at a point, it can only have **one**. Every defined x has exactly one output. **✅ Function.**

---

**Case 4: A circle (x² + y² = 25)**
```
   y
   |
 5 +    ___
   |   /   \
   |  |     |
   |  |     |
   |   \___/
-5 +
   |
───+────────→ x
```
A vertical line through x = 3 hits the circle at both y = 4 and y = -4 — two outputs from one input. **❌ Not a function.**

---

**Verbal statement:**
Imagine every possible vertical line sweeping across the graph. Each must touch the curve **at most once**. If even one vertical line hits twice — not a function.

---

## 5. One-to-One Functions and the Horizontal Line Test

A function is **one-to-one** (1-1) if no two different inputs produce the same output.

**Test:** Imagine every horizontal line sweeping across the graph. If any horizontal line hits the curve more than once, the function exists but is **not** one-to-one.

```
   Parabola y = x²:
   y
   |
4  + - - - - - -⊙- - - -⊙   ← horizontal line hits twice
   |       /   \ /
   |      /     \
───+─────────────→ x
```

The parabola **fails** the horizontal line test → not one-to-one. However it still passes the vertical line test → it **is** a function.

> One-to-one functions are critical later for **inverse functions**. Only one-to-one functions have inverses that are also functions.

## 6. Circles — A Non-Function and Its Pieces

### The equation

```
x² + y² = 25
```

Circle centred at origin, radius = 5.

Solving for y:

```
y² = 25 - x²
y  = ±√(25 - x²)
```

The ± sign is the problem — one x gives you two y-values. Not a function.

### Splitting into pieces

However, we can define **each half** as its own function:

```
Top half:     f(x) =  √(25 - x²)     [upper semicircle]
Bottom half:  g(x) = -√(25 - x²)     [lower semicircle]
```

```
   y         Top half: f(x)
   |
 5 +    ___
   |   /   \
   |  /     \
───+──────────→ x
  -5          5


   y         Bottom half: g(x)
   |
───+──────────→ x
  -5          5
   |  \     /
   |   \___/
-5 +
   |
```

Each piece individually passes the vertical line test → each is a function.

> **Key insight:** The notation `y = √(...)` always implies the *positive* square root only. The ± only appears when you introduce a square root to both sides of an equation.

## 7. Piecewise Functions

### What is a piecewise function?

A piecewise function is one whose formula **changes** depending on the value of the input x. Different rules apply across different intervals of the domain.

```
         ⎧  formula₁    for x in range₁
f(x)  =  ⎨  formula₂    for x in range₂
         ⎩  formula₃    for x in range₃
```

> The curly brace signals "all these pieces together make one function."

### The Absolute Value Function — The Classic Piecewise

```
         ⎧  x     if x ≥ 0
|x|  =   ⎨
         ⎩  -x    if x < 0
```

- If x is positive or zero: leave it alone
- If x is negative: multiply by -1 to flip the sign

Why? Distance is never negative. |−12| = −(−12) = 12.

**Graph:**
```
   y
   |  \   /
   |   \ /
   |    V
───+────────→ x
       0
```

The V-shape comes directly from these two pieces:
- Right side (x ≥ 0): y = x → slope of +1 through origin
- Left side (x < 0): y = -x → slope of -1 through origin
- Vertex at origin (0, 0)

### How to Graph Any Piecewise Function

**Step-by-step process:**
1. Identify the **key x-values** where the formula switches (the "boundary" values)
2. Mark those values on a number line to see your intervals
3. For each interval, **graph that piece as if it were a standalone function**
4. **Erase** (or ignore) any part of that piece outside its assigned interval
5. At boundary points:
   - **Closed circle ●** if the boundary is included (≤ or ≥)
   - **Open circle ○** if the boundary is excluded (< or >)
6. Never let two pieces overlap at the same x-value (that would fail the function test)

### Worked Example: Three-Piece Function

```
         ⎧  0              if x ≤ -1
f(x)  =  ⎨  √(1 - x²)     if -1 < x < 1
         ⎩  x              if x ≥ 1
```

**Step 1 — Identify intervals:**
- Piece 1: x ≤ -1 (left of -1)
- Piece 2: -1 < x < 1 (between -1 and 1)
- Piece 3: x ≥ 1 (right of 1)

**Step 2 — Graph each piece:**

**Piece 1: y = 0 for x ≤ -1**
This is a horizontal line at y = 0 (the x-axis), but only for x ≤ -1.
```
y
|
+━━━━●──────────────────  (closed at x = -1, y = 0)
     -1
```

**Piece 2: y = √(1 - x²) for -1 < x < 1**
Square both sides: y² = 1 - x² → x² + y² = 1 (a circle, radius = 1, centred at origin).
Since we have the positive square root only, this is the **top half** of the unit circle.
Defined only between x = -1 and x = 1, but with **open circles** at both ends.
```
y
|
1 +      ⌢        (semicircle arc from -1 to 1, open endpoints)
|      ○   ○
|    -1     1
```

The left open circle is "filled in" by Piece 1 (which ends at x = -1 with a closed circle) — perfect, no overlap.

**Piece 3: y = x for x ≥ 1**
This is the diagonal line y = x, but only for x ≥ 1.
```
y
|          /
|         /
|        ● (closed at x=1, y=1)
|       /
|      /
───────+──────→ x
       1
```

**Combined graph:**
```
   y
 1 +      ⌢
   |  ━━━●   ●──/
   |             /
   |            /
───+─────────────→ x
  -1    0    1
```

Reading left to right:
- Horizontal line at y = 0 for x ≤ -1 (closed dot at x = -1)
- Top semicircle of unit circle for -1 < x < 1 (open ends)
- Diagonal line y = x from x = 1 upward (closed dot at x = 1)

> The three pieces connect seamlessly — no gaps, no overlaps, one coherent function.

## 8. Domain and Natural Domain

### Definitions

**Domain:** all values you can input into a function and receive a valid output.

**Range:** all values that can come out of a function as outputs.

```
Domain  →  [function]  →  Range
(inputs)                  (outputs)
```

### Natural Domain

The **natural domain** is everything that works in a formula, automatically taking into account any **formulaic restrictions** (division by zero, square roots of negatives) and any **realistic constraints** (physical or contextual).

```
Natural domain = all values where the formula is defined
```

> Unless explicitly restricted, when we say "domain" in Calculus we mean the natural domain.

### Restrictions come from two sources only:

| Problem type | Rule |
|-------------|------|
| **Denominator** | Set denominator ≠ 0 → solve for excluded x-values |
| **Even root (√, ⁴√, etc.)** | Set radicand ≥ 0 → solve for valid x-values |

No other standard algebraic operation restricts the domain (polynomials, odd roots, multiplication, addition are defined everywhere).

---

## 9. Finding Domain: Techniques and Examples

### Example 1 — Polynomial (no restrictions)

```
f(x) = x³
```

Polynomials are defined for all real numbers. No denominators, no roots.

```
Domain:  all real numbers  →  (-∞, ∞)
```

Or symbolically: x ∈ ℝ

---

### Example 2 — Rational function (denominator)

```
         x + 1
f(x) = ——————————
       (x-1)(x-3)
```

**Method:** Set denominator ≠ 0:

```
(x - 1)(x - 3) ≠ 0
x - 1 ≠ 0    and    x - 3 ≠ 0
x ≠ 1         and    x ≠ 3
```

```
Domain:  all real numbers, x ≠ 1 and x ≠ 3
```

In interval notation:  (-∞, 1) ∪ (1, 3) ∪ (3, ∞)

---

### Example 3 — Tangent function

```
f(x) = tan x  =  sin x / cos x
```

tan x is undefined wherever cos x = 0.

cos x = 0 at:  x = π/2 + kπ   for any integer k

```
Domain:  x ≠ π/2 + kπ,  k ∈ ℤ
```

This is why tan x has those periodic vertical "jumps" — it's undefined at each of those x-values and the graph shoots to ±∞.

---

### Example 4 — Square root with quadratic (sign analysis)

```
f(x) = √(x² - 5x + 6)
```

**Step 1 — Set radicand ≥ 0:**

```
x² - 5x + 6 ≥ 0
```

**Step 2 — Factor:**

```
(x - 2)(x - 3) ≥ 0
```

**Step 3 — Find critical (boundary) points:**

Set each factor = 0:  x = 2  and  x = 3

**Step 4 — Sign analysis (number line method):**

Place critical points on a number line — they slice it into 3 intervals:

```
────────────●────────────●────────────
         x = 2        x = 3

Interval 1:  x < 2         (test x = 0)
Interval 2:  2 < x < 3     (test x = 2.5)
Interval 3:  x > 3         (test x = 4)
```

Test each interval in the expression `(x-2)(x-3)`:

| Interval | Test value | (x-2) | (x-3) | Product | Sign |
|----------|-----------|-------|-------|---------|------|
| x < 2 | x = 0 | − | − | + | ✅ positive |
| 2 < x < 3 | x = 2.5 | + | − | − | ❌ negative |
| x > 3 | x = 4 | + | + | + | ✅ positive |

We need the expression ≥ 0, so we want the **positive** intervals, plus the boundary points themselves (where the expression = 0):

```
Domain:  (-∞, 2] ∪ [3, ∞)
```

Brackets at 2 and 3 because the radicand is allowed to equal zero (√0 = 0, which is defined).

**Why you cannot just split into two inequalities:**

If you naively set each factor ≥ 0 separately:
```
x - 2 ≥ 0  →  x ≥ 2
x - 3 ≥ 0  →  x ≥ 3
```
This gives x ≥ 2 (wrong) and x ≥ 3 (wrong). It misses the logic that a product of **two negatives is positive**. Always use the sign analysis test for quadratic inequalities.

---

### Example 5 — Square root (simple)

```
f(x) = √(x - 1)
```

Set radicand ≥ 0:

```
x - 1 ≥ 0
x ≥ 1
```

```
Domain:  [1, ∞)
```

---

### Example 6 — Rational + simplification trap

```
       x² - 4
f(x) = ———————
        x - 2
```

**Step 1 — Set denominator ≠ 0:**

```
x - 2 ≠ 0   →   x ≠ 2
```

**Step 2 — Simplify:**

```
x² - 4 = (x+2)(x-2)

       (x+2)(x-2)
f(x) = ———————————  =  x + 2    (after cancelling x - 2)
          x - 2
```

The simplified form `x + 2` looks like it has no restrictions — but **you cannot cancel away the original domain restriction!**

```
Domain:  all real numbers, x ≠ 2
```

The cancellation reveals a **hole** (removable discontinuity) at x = 2, not a repair of the domain. The point (2, 4) is missing from the graph.

> **Rule:** You can simplify your function, but you must **keep the original domain** determined before simplification.

## 10. Finding Range

Range is harder to find in general. Two methods:

### Method 1 — Plug in domain, observe outputs

For `f(x) = √(x - 1)` with domain [1, ∞):

- Smallest input: x = 1 → f(1) = √0 = 0
- As x increases toward ∞ → f(x) increases toward ∞

```
Range:  [0, ∞)
```

### Method 2 — Solve for x in terms of y

For:
```
       x + 2
f(x) = ———————
        x - 1
```

Let y = f(x), then solve for x:

```
y(x - 1) = x + 2
xy - y = x + 2
xy - x = y + 2
x(y - 1) = y + 2
       y + 2
x  =  ———————
       y - 1
```

Now find the domain of **this expression** (treating y as the variable):

```
y - 1 ≠ 0   →   y ≠ 1
```

The range of the original f(x) is all real numbers except y = 1.

```
Range:  all real y, y ≠ 1
```

This also reveals that y = 1 is a **horizontal asymptote** of the original function.

> Note: This method works cleanly for rational functions but becomes difficult for more complex expressions. Better techniques using **limits** will come in Chapter 1.

## 11. Holes vs. Vertical Asymptotes

When a denominator causes a domain problem, there are two possibilities:

### How to classify:

Plug the excluded x-value into the original function:

| What you get | Type |
|-------------|------|
| **0 / 0** | Hole (removable discontinuity) — you can factor and cancel |
| **nonzero / 0** | Vertical asymptote — you cannot cancel it |

### Hole (Removable Discontinuity)

Example:
```
       x² - 4       (x+2)(x-2)
f(x) = ———————  =   ——————————
        x - 2           x - 2
```

Plug in x = 2:  (4 - 4)/(2 - 2) = **0/0** → hole

After cancelling: f(x) = x + 2, undefined at x = 2

Graph: the straight line y = x + 2 with a **hole** (open circle) at x = 2, y = 4.

```
   y
   |        /
 4 +   - - ○  ← hole at (2, 4)
   |      /
   |     /
   |    /
───+───────────→ x
   0   2
```

### Vertical Asymptote

Example:
```
        5
f(x) = ———————
        x - 4
```

Plug in x = 4:  5/(4 - 4) = **5/0** → vertical asymptote

Cannot be cancelled. The function blows to ±∞ near x = 4.

```
   y
   |          |
   |          |  ← vertical asymptote at x = 4
   | ____     |
   |          |
   |          |_____
───+──────────+──────→ x
              4
```

**Quick Summary:**
```
Plug in bad x-value:
  → get 0/0        = HOLE          (removable)
  → get number/0   = ASYMPTOTE     (non-removable)
```

## 12. Real-World Domain: The Box Problem

### Problem setup

Take a flat sheet of cardboard **16 inches × 30 inches**. Cut equal squares (side length x) from each corner. Fold up the sides to form an open box.

```
┌─────────────────────────────────┐
│  ┌──┐                    ┌──┐  │  ← cut squares of side x
│  └──┘                    └──┘  │
│                                 │   30 in total length
│                                 │
│  ┌──┐                    ┌──┐  │
│  └──┘                    └──┘  │
└─────────────────────────────────┘
         16 in total width
```

After cutting and folding:
- Length becomes: 30 - 2x  (lose one x on each end)
- Width becomes: 16 - 2x
- Height becomes: x (the square you folded up)

### Volume formula

```
V(x) = x · (16 - 2x) · (30 - 2x)
```

### Finding the natural domain

**Step 1 — Formulaic checks:**
- No denominators → no issues there
- No roots → no issues there

**Step 2 — Realistic constraints:**
- x cannot be negative: you cannot make a negative-length cut → **x > 0**
- x cannot be zero (trivially: flat box with no height — not really a useful box, but mathematically could be included)
- x has a maximum: you are cutting from the **16-inch side**, and each end loses x. To avoid the cuts overlapping: 2x < 16 → **x < 8**
  - At x = 8 exactly, you cut the entire 16-inch side in half and the box collapses

```
Domain:  0 < x < 8     (open interval)
```

Or for strict mathematical completeness: 0 ≤ x ≤ 8 (degenerate boxes with zero volume at the boundaries are technically possible but physically meaningless).

> **Lesson:** Real-world problems can have domain restrictions that no formula would reveal. Always apply physical/contextual reasoning in addition to checking denominators and roots.

## 13. Odd and Even Functions

### Even Functions

**Definition:** f(-x) = f(x) for all x in the domain.

Plugging in a negative x gives the **same result** as plugging in the positive x.

**Geometrically:** symmetric across the **y-axis** (mirror image left–right).

**Test:** Replace every x with (-x). Simplify. If you get back the original f(x) → even.

**Example:** f(x) = x⁴ - x² + 1

```
f(-x) = (-x)⁴ - (-x)² + 1
      = x⁴ - x² + 1
      = f(x)     ✅ EVEN
```

**Graph (even symmetry):**
```
   y
   |
   |    _   _
   |   / \ / \
   |__/   V   \__
───+───────────────→ x
```

Mirror image across y-axis.

---

### Odd Functions

**Definition:** f(-x) = -f(x) for all x in the domain.

Plugging in a negative x gives the **negative** of what you'd get from the positive.

**Geometrically:** symmetric about the **origin** (180° rotational symmetry).

**Test:** Replace every x with (-x). Simplify. If every term flips sign and you can factor out -1 to get -f(x) → odd.

**Example:** g(x) = x³ - x

```
g(-x) = (-x)³ - (-x)
      = -x³ + x
      = -(x³ - x)
      = -g(x)     ✅ ODD
```

**Graph (odd symmetry):**
```
   y
   |  /
   | /
   |/
───+────→ x
   |    \
   |     \
```

If you rotate 180° around the origin, the graph maps onto itself.

---

### Neither Even Nor Odd

Most functions are neither. If f(-x) ≠ f(x) and f(-x) ≠ -f(x), the function has no special symmetry.

**Example:** f(x) = x² + x

```
f(-x) = (-x)² + (-x) = x² - x    ← not equal to f(x), not equal to -f(x)
```

Neither even nor odd.

---

### Summary Table

| Type | Algebraic condition | Graph symmetry |
|------|--------------------|--------------------|
| Even | f(-x) = f(x) | Symmetric across y-axis |
| Odd | f(-x) = -f(x) | Symmetric about origin (180° rotation) |
| Neither | Neither condition holds | No special symmetry |

---

### Typical patterns (useful for quick recognition)

| Function | Type |
|---------|------|
| y = xⁿ, n even (x², x⁴, ...) | Even |
| y = xⁿ, n odd (x, x³, x⁵, ...) | Odd |
| y = cos x | Even |
| y = sin x | Odd |
| y = tan x | Odd |
| y = |x| | Even |
| y = x² + x | Neither |

## 14. Key Concept Summary

```
FUNCTION
  Each input → exactly one output
  Tested graphically: Vertical Line Test

ONE-TO-ONE
  Each output ← exactly one input
  Tested graphically: Horizontal Line Test

NATURAL DOMAIN
  All x-values that produce a defined output
  Check: denominators ≠ 0, radicands of even roots ≥ 0

DOMAIN ISSUES
  Denominators:  set denominator = 0 → excluded points
  Even roots:    set radicand ≥ 0 → sign analysis

HOLES vs ASYMPTOTES
  Plug bad x in: 0/0 → hole (removable)
                 k/0 → vertical asymptote (non-removable)

PIECEWISE
  Different formulas for different intervals
  Graph piece by piece using correct interval, open/closed circles

EVEN:  f(-x) = f(x)   → y-axis symmetry
ODD:   f(-x) = -f(x)  → origin symmetry
```

## 15. Embodied AI Connections

The function concept is not just mathematical abstraction — it is the **computational DNA** of every Embodied AI system. Every sensor, actuator, control algorithm, and neural network is, at its core, a function or a composition of functions.

---

### 15.1 The Robot as a Function

At the highest level, a robot is a function:

```
State × Input  →  New State × Output

f(current_position, sensor_data, motor_command) → next_position
```

The function requirement — **one specific output per input** — is essential. If a motor controller could produce two different torques for the same command, the robot would be unpredictable and uncontrollable.

---

### 15.2 Sensor Models — Functions with Restricted Domain

Every physical sensor has a valid operating range. A LiDAR sensor might measure distances from 0.1 m to 30 m. An IMU might saturate beyond a certain angular rate. These are **domain restrictions** of the sensor's function.

```
LiDAR:  d = f(θ)     domain: θ ∈ [−60°, 60°],  output range: [0.1, 30.0] m
```

Outside the domain, the sensor output is undefined or unreliable. The robotics engineer must model and handle these restrictions explicitly — exactly as we find the natural domain in Section 0.2.

---

### 15.3 Piecewise Functions in Control Systems

Real robot controllers constantly use piecewise logic:

**Velocity control with zones:**
```
         ⎧  v_max          if error > threshold
v  =     ⎨  k · error      if |error| ≤ threshold    (proportional zone)
         ⎩  -v_max         if error < -threshold
```

This is a piecewise function of the error signal. The "formula switches" depending on whether the robot is far from, close to, or past the target. This is called **saturated proportional control** or a **bang-bang-with-deadzone** controller.

**Friction models in simulation (Stribeck curve):**
Friction force as a function of velocity has distinct regimes (static, Stribeck, viscous) — each requiring a different sub-formula. This is directly a piecewise function modelling a physical phenomenon.

---

### 15.4 Domain Restrictions in Inverse Kinematics

In robotic arm control, the joint configuration space has strict domain limits:
- Joint angles are mechanically bounded: θ ∈ [θ_min, θ_max]
- Some configurations cause singularities (akin to division by zero) where the Jacobian matrix loses rank and inverse kinematics breaks down

Identifying these singularities is equivalent to finding domain restrictions in a complex multivariable function. The natural domain of the IK solution excludes singular configurations — just as we exclude x = 0 from f(x) = 1/x.

---

### 15.5 Neural Networks as Function Approximators

A neural network is, formally, a parameterised function:

```
f_θ: ℝⁿ → ℝᵐ
```

Where:
- n = dimension of input (e.g., sensor readings, image pixels)
- m = dimension of output (e.g., motor commands, predicted labels)
- θ = learnable weights

The function must satisfy the definition: **each input vector maps to exactly one output vector**. For the same input state, the network always produces the same output (given fixed weights) — deterministic function behaviour.

**Activation functions** (ReLU, sigmoid, tanh) are all piecewise or otherwise restricted functions:

```
ReLU:   f(x) = ⎧  x    if x ≥ 0
               ⎩  0    if x < 0

Sigmoid:  f(x) = 1 / (1 + e^(-x))     (domain: all reals; range: (0, 1))
```

ReLU is literally a piecewise function of the type introduced in Section 0.2.

---

### 15.6 Odd and Even Symmetry in Signal Processing

In robotics, signals from IMUs, encoders, and force/torque sensors are processed mathematically. Odd and even function concepts are exploited in:

- **Fourier series decomposition** — any signal can be decomposed into a sum of odd (sine) and even (cosine) functions. This underlies all frequency-domain filtering of sensor noise.
- **Filter design** — symmetric (even) filters introduce no phase shift (zero-phase filtering); anti-symmetric (odd) filters are used for derivative estimation. In robotics, zero-phase filters preserve the timing of events in sensor data, which matters for low-latency control loops.
- **Symmetry exploitation in learning** — if the robot environment has symmetrical properties (bilateral locomotion, mirrored arm kinematics), enforcing even/odd symmetry constraints on learned policies can drastically reduce sample complexity.

---

### 15.7 Vertical and Horizontal Asymptotes — Physical Limits

Asymptotes model physical boundaries that systems approach but cannot reach:

- **Vertical asymptotes** appear in control: as a robotic arm approaches a singular configuration, the required joint velocities grow without bound (→ ∞). This is a vertical asymptote of the velocity function and signals a physical impossibility.
- **Horizontal asymptotes** appear in dynamics: the terminal velocity of a falling drone under aerodynamic drag asymptotically approaches a finite limit — the horizontal asymptote of the velocity-versus-time function.

Understanding asymptotic behaviour — learned first here through domain analysis — is essential for safe robot operation.
