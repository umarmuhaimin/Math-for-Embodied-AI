## 1. Angles: Initial Side, Terminal Side, and Direction

### Anatomy of an Angle

An angle is formed by rotating a ray from a starting position to an ending position, measured relative to the **positive x-axis**.

```
        terminal side
       /
      /
     / ) θ
────+──────────────→  x-axis (initial side)
```

| Term | Meaning |
|------|---------|
| **Initial side** | The starting ray — always the positive x-axis |
| **Terminal side** | The ending ray — where the rotation stops |
| **θ (theta)** | The measure of the angle between them |

### Direction of Rotation

```
          +θ (counterclockwise)
          ↑
          |   ↗
          |  /
──────────+──────────→  x
          |
    ↙     |
 −θ (clockwise)
```

- **Counterclockwise** rotation → **positive** angle
- **Clockwise** rotation → **negative** angle

> This sign convention is universal in mathematics and physics. A motor spinning counterclockwise produces positive angular displacement; clockwise produces negative.

## 2. Degrees and Radians

### Two Systems for Measuring Angles

Both systems describe the same angles — just different units.

| System | Full circle | Intuition |
|--------|------------|-----------|
| Degrees | 360° | Familiar, everyday use |
| Radians | 2π | Mathematical, relates arc length to radius |

### Fundamental Equality

One full circle in both systems:

```
2π radians  =  360°
```

Dividing both sides by 2:

```
π radians  =  180°
```

This single equality is all you need to convert between systems.

### Conversion Formulas

**Degrees → Radians:** multiply by π/180

```
radians  =  degrees × π/180
```

**Radians → Degrees:** multiply by 180/π

```
degrees  =  radians × 180/π
```

### Memory trick

> If you want to **remove degrees** (convert to radians), put degrees in the denominator: use **π/180** (degrees cancel with the unit in the denominator).  
> If you want to **remove radians** (convert to degrees), put π in the denominator: use **180/π**.

### Worked Examples

**Example 1 — 200° → radians:**

```
200° × π/180  =  200π/180  =  10π/9 radians
```

**Example 2 — −3π/4 radians → degrees:**

```
−3π/4 × 180/π  =  −3 × 180/4  =  −540/4  =  −135°
```

### Common Angle Reference Table

| Degrees | Radians |
|---------|---------|
| 0° | 0 |
| 30° | π/6 |
| 45° | π/4 |
| 60° | π/3 |
| 90° | π/2 |
| 120° | 2π/3 |
| 135° | 3π/4 |
| 150° | 5π/6 |
| 180° | π |
| 270° | 3π/2 |
| 360° | 2π |

## 3. Graphing Angles on the Unit Circle

### The Four Cardinal Points

```
              π/2 (90°)
               |
               |
 π (180°) ─────+───── 0 / 2π (0° / 360°)
               |
               |
             3π/2 (270°)
```

### How to Graph Any Angle

**Method:** Divide each half-circle (π) into the number of sections equal to the denominator of your angle. Count the numerator of sections.

**Example: graph 4π/3**

- Denominator is 3 → divide each π into 3 equal parts → 6 sections total around the circle
- Numerator is 4 → count 4 sections counterclockwise from the initial side

```
              π/2
               |
       2π/3   |   π/3
          \   |   /
           \  |  /
   π ───────\─+─/──────── 0
            /\|/\
           /  |  \
       4π/3   |   5π/3    ← 4π/3 lands here (3rd quadrant)
               |
             3π/2
```

**Example: graph 5π (multiple revolutions)**

- Count 5 half-circles: 1π → 2π → 3π → 4π → 5π
- 5π = 2(2π) + π → two full revolutions plus half a revolution → lands at π (same as 180°)

**Example: graph −7π/2 (negative angle)**

- Negative → clockwise rotation
- Break each π into 2 parts; count 7 clockwise
- 7π/2 clockwise = 3 full revolutions + π/2 clockwise → lands at 3π/2 reference point (going clockwise from 0)

## 4. Co-terminal Angles

Two angles are **co-terminal** if they have the **same terminal side** despite having different measures.

```
−2π/3  and  4π/3  are co-terminal

(going clockwise 2π/3)  ≡  (going counterclockwise 4π/3)
Both terminal sides point to the same location.
```

**General formula:** angles θ and θ + 2πk are co-terminal for any integer k.

```
5π/3  and  5π/3 − 2π = −π/3  are co-terminal
```

> You can always add or subtract multiples of 2π without changing the terminal side.

## 5. The Unit Circle and Trig Functions

### What is the Unit Circle?

A circle centred at the origin with **radius = 1**.

```
         (0,1)
          |
(-1,0) ───+─── (1,0)
          |
         (0,-1)
```

Every point on the unit circle has coordinates **(cos θ, sin θ)** for the angle θ from the positive x-axis.

### Why the Unit Circle Works

For any angle θ, form a right triangle inside the unit circle:

```
          (x, y)
         /|
      1 / |  y  (opposite)
       /  |
      / θ |
─────+────+────
     x (adjacent)
```

- Hypotenuse = 1 (radius of unit circle)
- Adjacent = x
- Opposite = y

From SOH CAH TOA:
```
sin θ = opposite/hypotenuse = y/1 = y
cos θ = adjacent/hypotenuse = x/1 = x
```

**Therefore, on the unit circle:**
```
cos θ  =  x-coordinate of the point
sin θ  =  y-coordinate of the point
```

The coordinate pair is always written **(cos θ, sin θ)** — cosine first (x), sine second (y).

### Unit Circle with Key Values

```
                 (0, 1)
                π/2
          2π/3 ╱  ╲ π/3
   (−½, √3/2) ╱    ╲ (½, √3/2)
              ╱      ╲
   3π/4 ─── ╱        ╲ ─── π/4
(−√2/2,√2/2)            (√2/2,√2/2)
             |          |
5π/6 ────────+──────────+──────── π/6
(−√3/2, ½)  │            │  (√3/2, ½)
             │            │
π ───────────+────────────+─────── 0 / 2π
(−1, 0)      │            │        (1, 0)
             │            │
7π/6 ────────+──────────+──────── 11π/6
(−√3/2,−½)  │            │  (√3/2,−½)
             │            │
  5π/4 ─── ╲             ╱ ─── 7π/4
(−√2/2,−√2/2)╲          ╱ (√2/2,−√2/2)
              ╲          ╱
  4π/3 ─────── ╲        ╱ ─────── 5π/3
(−½,−√3/2)      ╲      ╱  (½,−√3/2)
                  3π/2
                 (0,−1)
```

**Must memorise at minimum:**
- 0, π/6, π/4, π/3, π/2 in Quadrant 1 (all values positive)
- The pattern repeats in other quadrants with sign changes (use ASTC)

## 6. The Six Trig Functions (SOH CAH TOA)

### Important: Trig functions are FUNCTIONS, not numbers

> **sin, cos, tan, csc, sec, cot are functions — they require an angle argument.**  
> Writing "sin" alone means nothing. You must write sin(θ).  
> The identity sin²θ + cos²θ = 1 only works when **both** sides carry the **same angle θ**.

### Primary Functions

```
SOH:   sin θ  =  opposite / hypotenuse  =  y/r  =  y (on unit circle)

CAH:   cos θ  =  adjacent / hypotenuse  =  x/r  =  x (on unit circle)

TOA:   tan θ  =  opposite / adjacent    =  y/x  =  sin θ / cos θ
```

### Reciprocal Functions

Each primary function has a reciprocal:

```
csc θ  =  1/sin θ  =  hypotenuse/opposite  =  1/y

sec θ  =  1/cos θ  =  hypotenuse/adjacent  =  1/x

cot θ  =  1/tan θ  =  adjacent/opposite    =  x/y
```

### Reciprocal Pairs (memorise these pairings)

| Primary | Reciprocal |
|---------|-----------|
| sin | csc |
| cos | sec |
| tan | cot |

> Trick: sin ↔ csc, cos ↔ sec, tan ↔ cot. Note that **co**sine pairs with **se**cant (not cosecant), and **co**tangent pairs with tangent. The "co" prefix does not mean they are reciprocals of each other.

### On the Unit Circle

| Function | Unit Circle expression |
|----------|----------------------|
| sin θ | y |
| cos θ | x |
| tan θ | y/x |
| csc θ | 1/y |
| sec θ | 1/x |
| cot θ | x/y |

## 7. ASTC: Signs Across Quadrants

### The Four Quadrants

```
         Q2  |  Q1
             |
  ───────────+───────────
             |
         Q3  |  Q4
```

### ASTC Rule — "All Students Take Calculus"

| Quadrant | What's positive |
|----------|----------------|
| **Q1 (All)** | All six functions positive |
| **Q2 (Students → Sin)** | sin, csc positive; rest negative |
| **Q3 (Take → Tan)** | tan, cot positive; rest negative |
| **Q4 (Calculus → Cos)** | cos, sec positive; rest negative |

```
    S | A             sin+ | All+
   ———+———    →      ──────+──────
    T | C            tan+  | cos+
```

### Why this makes sense geometrically

- **Q1:** x > 0, y > 0 → all ratios positive
- **Q2:** x < 0, y > 0 → sin = y/r > 0; cos = x/r < 0; tan = y/x < 0
- **Q3:** x < 0, y < 0 → sin < 0; cos < 0; tan = y/x > 0 (negative/negative)
- **Q4:** x > 0, y < 0 → sin < 0; cos > 0; tan < 0

## 8. Reference Angles (Critical Angles)

### Definition

A **reference angle** (also called a **critical angle**) is the **acute** angle between the terminal side of your angle and the **x-axis** (not the y-axis).

> The reference angle is always positive and always between 0 and π/2.  
> **All trig computations start with the reference angle (Quadrant 1), then the sign is determined by ASTC.**

### The Four Cases

**Case 1 — Q1: angle is already acute**

```
    |  /
    | / ← θ (acute)
    |/____
```

Reference angle = θ (unchanged — already in Q1)

---

**Case 2 — Q2:**

```
    |\ 
    | \  ← terminal side
    |  \
────+────
    | ref
    |  angle
```

The angle goes past π/2. The reference angle is the gap between the terminal side and the x-axis:

```
Reference angle  =  π − θ
```

---

**Case 3 — Q3:**

```
    |
────+────
    |\
    | \  ← terminal side (past π)
    |  \ref
```

The angle is past π. The reference angle is how far past π we've gone:

```
Reference angle  =  θ − π
```

---

**Case 4 — Q4:**

```
    |   /  ref
────+──/───────
    | /  ← terminal side
    |/
```

The angle is between 3π/2 and 2π. The reference angle is what remains to reach 2π:

```
Reference angle  =  2π − θ
```

### Summary Table

| Quadrant | Reference angle formula |
|----------|------------------------|
| Q1 | θ (already acute) |
| Q2 | π − θ |
| Q3 | θ − π |
| Q4 | 2π − θ |

> **Critical rule:** The reference angle must always be acute (between 0 and π/2). If your answer is not acute, you've used the wrong formula or the wrong quadrant.

## 9. Using Reference Angles to Find Any Trig Value

### The Two-Step Process

1. **Find the reference angle** using the formulas above — this gives you the magnitude of the trig value (always positive, always Q1)
2. **Apply the ASTC sign** based on which quadrant the original angle is in

> Trig values from the reference angle are always positive. The quadrant determines the final sign.

### Worked Example: Find all six trig functions of 5π/3

**Step 1 — Graph the angle:**

5π/3 → denominator 3, so divide each π into 3 parts → 6 sections.  
Count 5 counterclockwise: lands just past 3π/2, in **Quadrant 4**.

```
              π/2
               |
               |
   π ──────────+──────────── 0
               |       \
               |        \  ← 5π/3 (Q4)
             3π/2
```

**Step 2 — Find the reference angle:**

In Q4:  reference angle = 2π − θ

```
2π − 5π/3  =  6π/3 − 5π/3  =  π/3
```

Reference angle = **π/3** ✅ (acute, in Q1)

**Step 3 — Find trig values of π/3 (Q1 values):**

```
sin(π/3)  =  √3/2
cos(π/3)  =  1/2
tan(π/3)  =  √3
```

**Step 4 — Apply ASTC for Q4 (only cos/sec positive):**

```
sin(5π/3)  =  −√3/2    (sin is negative in Q4)
cos(5π/3)  =  +1/2     (cos is positive in Q4)
tan(5π/3)  =  −√3      (tan is negative in Q4)
csc(5π/3)  =  −2/√3    (reciprocal of sin, negative)
sec(5π/3)  =  +2       (reciprocal of cos, positive)
cot(5π/3)  =  −1/√3    (reciprocal of tan, negative)
```

### The Reference Angle Workflow

```
Given angle θ
       ↓
Locate quadrant
       ↓
Compute reference angle (acute, Q1 formula)
       ↓
Find sin/cos/tan of reference angle (always positive)
       ↓
Apply ASTC sign for the original quadrant
       ↓
Final answer ✓
```

## 10. Essential Trig Identities

These must be memorised — they appear in every calculus chapter.

### Fundamental Identities

```
tan θ  =  sin θ / cos θ
cot θ  =  cos θ / sin θ
```

### Pythagorean Identities

```
sin²θ + cos²θ  =  1          ← the primary one

1 + tan²θ      =  sec²θ      (divide primary by cos²θ)

1 + cot²θ      =  csc²θ      (divide primary by sin²θ)
```

**Derived forms (rearrangements — very useful in integration):**

```
sin²θ  =  1 − cos²θ
cos²θ  =  1 − sin²θ
```

### Double Angle Formulas

```
sin(2θ)  =  2 sin θ cos θ
cos(2θ)  =  cos²θ − sin²θ
          =  1 − 2sin²θ
          =  2cos²θ − 1
```

### Half Angle Formulas (derived from double angle)

```
sin²θ  =  (1 − cos 2θ) / 2
cos²θ  =  (1 + cos 2θ) / 2
```

> These are used directly in integration of sin²x and cos²x — critical for Calculus 2.

### Reciprocal Identities

```
csc θ  =  1/sin θ
sec θ  =  1/cos θ
cot θ  =  1/tan θ
```

## 11. Graphing Trig Functions: The Standard Forms

### Full General Form

```
y  =  a · sin(bx − c) + d
y  =  a · cos(bx − c) + d
```

Each parameter does something independent:

| Parameter | Effect | What changes |
|-----------|--------|-------------|
| **a** | Amplitude | Vertical stretch/shrink, flip if negative |
| **b** | Period | Horizontal stretch/shrink |
| **c** | Phase shift | Horizontal translation (left/right) |
| **d** | Vertical shift | Vertical translation (up/down) |

> **Key insight:** a and b affect the shape/size of the graph. c and d only move it — they have **no effect on amplitude or period**.

---

## 12. Amplitude: The A Parameter

### Definition

**Amplitude = |a|**

The amplitude is always **positive**. It measures how far the graph reaches above and below the x-axis (or the midline if d ≠ 0).

```
y_max  =  |a|
y_min  =  −|a|
Total height of one cycle = 2|a|
```

### Effect of the sign of a

| Sign of a | Effect |
|-----------|--------|
| a > 0 | Graph behaves in its natural direction |
| a < 0 | Graph is **flipped** (reflected across x-axis) |

**Example:**

```
y =  2 sin x  →  amplitude 2, natural orientation
y = −2 sin x  →  amplitude 2, flipped (starts going DOWN from 0)
```

```
y = 2 sin x:        y = −2 sin x:
    2 |  ∿           2 |
      |               |
──────+──────    ──────+──────
      |               |    ∿
   −2 |            −2 |
```

### Stretch vs Shrink

| |a| value | Effect |
|----------|--------|
| |a| > 1 | Amplitude gain — graph taller |
| 0 < |a| < 1 | Amplitude shrink — graph flatter |

---

## 13. Period: The B Parameter

### Definition

```
Period  =  2π / b
```

The period is how long one complete cycle takes. The base period for sin and cos is 2π (one full revolution of the unit circle).

### Effect of b

| b value | Period | Effect |
|---------|--------|--------|
| b > 1 | Less than 2π | **Horizontal compression** (more cycles in same space) |
| 0 < b < 1 | More than 2π | **Horizontal stretch** (fewer cycles in same space) |
| b = 1 | 2π | Standard period |

```
b > 1 → compressed → more cycles per 2π window
b < 1 → stretched  → fewer cycles per 2π window
```

**Example: y = 2 sin(4x)**

```
Period = 2π/4 = π/2
```

This means one full cycle completes in π/2. In the standard 0 to 2π window, there are **4 complete cycles**.

**Example: y = −3 cos(0.5x)**

```
Period = 2π/0.5 = 4π
```

One full cycle completes in 4π — the graph is stretched out.

## 14. Phase Shift: The C Parameter

### Extracting the Phase Shift

The phase shift is not simply c. You must **factor b out** of the parentheses first:

```
y = a · sin(bx − c)

Factor b out:

y = a · sin[b(x − c/b)]
```

Now the shift is clearly **c/b** inside `(x − c/b)`.

### Direction of Shift

```
(x − c/b)  →  shift RIGHT by c/b
(x + c/b)  →  shift LEFT  by c/b
```

**Why does minus mean right?**  
A translation is defined as `(x − h)` where h is the shift. So `(x − c/b)` moves the graph to the **right** by c/b. An addition `(x + c/b)` is really `(x − (−c/b))`, which is a shift **left** by c/b.

**Timeline analogy:**  
Think of x as time. Adding to x means the event happens *sooner* (to the left on a timeline). Subtracting means the event happens *later* (to the right).

```
(x + k): event happens sooner → graph shifts LEFT
(x − k): event happens later  → graph shifts RIGHT
```

### Important: Phase shift does NOT affect amplitude or period

Phase shift only moves the graph horizontally. The shape, height, and cycle length are unchanged.

---

## 15. Vertical Shift: The D Parameter

```
y  =  a · sin(bx − c) + d
```

The **d** parameter shifts the entire graph **up or down**:

```
d > 0  →  graph shifts UP by d
d < 0  →  graph shifts DOWN by |d|
```

The **midline** of the graph moves from y = 0 to y = d.

```
Without d:  oscillates between −|a| and +|a|
With d:     oscillates between d − |a| and d + |a|
```

### Important: Vertical shift does NOT affect amplitude or period

d only moves the graph vertically. The shape and cycle length are unchanged.

## 16. The Five-Point Graphing Method

This is the systematic approach to sketch any sin or cos function. It works by identifying 5 key x-values within one complete period, then reading off y-values.

### Step 1 — Extract parameters

From `y = a · sin(b(x − c/b)) + d`:
- Amplitude: |a|, note sign of a for flip
- Period: T = 2π/b
- Phase shift: c/b (left if +, right if −)
- Vertical shift: d

### Step 2 — Identify the 5 key x-values

Starting from the phase shift (or 0 if no shift), divide one full period T into 4 equal quarters:

```
x₁ = start           (= phase shift, or 0)
x₂ = start + T/4
x₃ = start + T/2
x₄ = start + 3T/4
x₅ = start + T       (= end of one period)
```

### Step 3 — Read off y-values using the key-point rule

The pattern at these 5 points depends on the function and the sign of a:

**For y = a · sin(bx) with a > 0:**

```
x₁: y = 0      (start at zero)
x₂: y = +|a|   (peak)
x₃: y = 0      (cross zero)
x₄: y = −|a|   (trough)
x₅: y = 0      (end at zero, one cycle complete)
```

**For y = a · cos(bx) with a > 0:**

```
x₁: y = +|a|   (start at peak)
x₂: y = 0      (cross zero)
x₃: y = −|a|   (trough)
x₄: y = 0      (cross zero)
x₅: y = +|a|   (end at peak, one cycle complete)
```

**If a < 0 (negative a — graph flipped):**  
Reverse peak and trough: where sin normally peaks, it now troughs, and vice versa.

```
y = −a · sin: start 0, trough, 0, peak, 0
y = −a · cos: start trough, 0, peak, 0, trough
```

### The Midpoint Rule (from Professor Leonard)

> **If you start and end at zero, you cross zero in the middle.**  
> **If you start and end at a peak, you peak in the middle.**  
> **If you start and end at a trough, you trough in the middle.**  
>
> The middle point always matches the start and end behaviour.

```
Pattern:
start=0, middle=0, end=0   → peaks at T/4, troughs at 3T/4
start=peak, middle=peak, end=peak → crosses at T/4 and 3T/4, troughs at T/2
start=trough, middle=trough, end=trough → crosses at T/4 and 3T/4, peaks at T/2
```

### Step 4 — Apply phase shift

If there is a phase shift c/b, shift every x-value in your 5-point table by c/b in the appropriate direction. The y-values stay the same — only the x-positions move.

### Step 5 — Apply vertical shift

After plotting all 5 points, shift every y-value up or down by d. The x-positions stay the same.

## 17. Worked Graphing Examples

### Example 1 — y = 2 sin(4x)

**Parameters:**
```
a = 2   →  amplitude = 2, no flip
b = 4   →  period = 2π/4 = π/2
c = 0   →  no phase shift
d = 0   →  no vertical shift
```

**5 Key x-values** (T = π/2, divide into 4 quarters of π/8):

```
x₁ = 0
x₂ = π/8
x₃ = π/4
x₄ = 3π/8
x₅ = π/2
```

**5 Key (x, y) points:**

```
(0,   0)     ← start at zero
(π/8, 2)     ← peak
(π/4, 0)     ← cross zero
(3π/8,−2)    ← trough
(π/2, 0)     ← end at zero
```

**Graph (one cycle):**

```
 y
 2 +          ∧
   |         / \
 0 +────────/───\────────→ x
   | 0  π/8 π/4 3π/8 π/2
-2 +              ∨
```

Four of these cycles fit in the standard 0 to 2π window.

---

### Example 2 — y = −3 cos(0.5x)

**Parameters:**
```
a = −3  →  amplitude = 3, graph FLIPPED (cos normally starts at peak, now starts at trough)
b = 0.5 →  period = 2π/0.5 = 4π
c = 0   →  no phase shift
d = 0   →  no vertical shift
```

**5 Key x-values** (T = 4π, quarters of π):

```
x₁ = 0
x₂ = π
x₃ = 2π
x₄ = 3π
x₅ = 4π
```

**y-values (flipped cos):**

```
(0,  −3)     ← starts at trough (flipped, not peak)
(π,   0)     ← cross zero
(2π, +3)     ← peak (middle = same as ends for flipped cos, i.e., the peak)
(3π,  0)     ← cross zero
(4π, −3)     ← end at trough
```

**Graph:**


The graph is wider (stretched horizontally) and starts at the trough because of the negative a.

## 18. Key Concept Summary

```
ANGLE ANATOMY
  Initial side  = positive x-axis
  Terminal side = where rotation ends
  Counterclockwise = positive angle
  Clockwise         = negative angle

CONVERSION
  Degrees → radians:  × π/180
  Radians → degrees:  × 180/π
  Full circle:  2π = 360°

CO-TERMINAL ANGLES
  Same terminal side, different measures
  θ  and  θ + 2πk  are co-terminal

UNIT CIRCLE
  radius = 1
  cos θ = x-coordinate
  sin θ = y-coordinate

SOH CAH TOA
  sin = opp/hyp    cos = adj/hyp    tan = opp/adj
  csc = hyp/opp    sec = hyp/adj    cot = adj/opp

ASTC (signs by quadrant)
  Q1: All +     Q2: Sin +     Q3: Tan +     Q4: Cos +

REFERENCE ANGLE (critical angle)
  Always acute (0 to π/2), always with x-axis
  Q1: θ        Q2: π−θ      Q3: θ−π      Q4: 2π−θ
  Step 1: find acute reference angle
  Step 2: apply ASTC sign for original quadrant

GENERAL FORM: y = a·sin(bx − c) + d
  |a| = amplitude (vertical stretch/shrink)
       a < 0 = flipped graph
  2π/b = period (horizontal stretch/shrink)
       b > 1 = compression   b < 1 = stretch
  c/b = phase shift (horizontal translation only)
       (x − c/b): shift RIGHT     (x + c/b): shift LEFT
  d   = vertical shift (translation only)

FIVE KEY POINTS (per cycle)
  sin: 0, peak, 0, trough, 0
  cos: peak, 0, trough, 0, peak
  Flip if a < 0

MIDPOINT RULE
  start=0 & end=0  →  middle=0
  start=peak & end=peak  →  middle=peak
```

## 19. Embodied AI Connections

Trigonometry is the mathematical language of **rotation, oscillation, and wave behaviour** — three phenomena that define how physical robots move through the world. Every joint angle, every oscillating signal, every periodic controller relies on the concepts in Section 0.3.

---

### 19.1 Joint Angles and the Unit Circle

Every revolute (rotating) joint in a robotic arm is an angle on the unit circle. The position of the joint's output link is exactly the unit circle coordinate:

```
x-component of link  =  L · cos(θ)
y-component of link  =  L · sin(θ)
```

where L is the link length and θ is the joint angle. This is the unit circle formula applied directly to kinematics. The full forward kinematics of a multi-link arm is a chain of these expressions — one per joint.

**Reference angle → joint limit checking:** A robot's joint limits are described as angular bounds. The reference angle concept (finding the equivalent acute angle and determining its quadrant) directly applies when mapping commanded angles to hardware encoder readings.

---

### 19.2 ASTC and Motor Sign Conventions

When a motor drives a joint, the direction of torque depends on the sign of the commanded signal. ASTC in robotics becomes:

- Positive torque in certain quadrants produces desired motion
- Negative torque in others produces the same physical effect

Controllers use quadrant detection (equivalent to ASTC) to determine whether to apply positive or negative control effort. For example, a back-drivable joint may require sign-aware friction compensation that changes formula by quadrant — exactly analogous to the ASTC rule.

---

### 19.3 Oscillatory Motion — Amplitude and Period in Physics

A robot's end-effector tracing a periodic path, a quadruped's leg cycle, a drone's rotor vibration — all are sinusoidal motions described by:

```
x(t)  =  A · sin(ωt + φ)
```

Where:
- A = amplitude (how far the end-effector swings)
- ω = angular frequency = 2π/T (related to the b parameter: ω = b)
- φ = phase (the c/b shift)

**Amplitude** determines the workspace coverage of the motion. Too large an amplitude drives joints into hardware limits (outside the natural domain). Too small produces insufficient reach.

**Period** (T = 2π/ω) determines gait speed in legged robots. The b > 1 compression corresponds to faster gait cycles; b < 1 stretch corresponds to slower, more deliberate motion.

---

### 19.4 Phase Shift in Gait Synchronisation

Legged robots require coordinated leg movements. Each leg follows a sinusoidal swing/stance cycle, but they must be offset in time from each other. This offset is the **phase shift** c/b.

For a quadruped trotting gait:
```
Front-left leg:   x_FL(t) = A · sin(ωt)
Front-right leg:  x_FR(t) = A · sin(ωt + π)    ← shifted left by π (anti-phase)
Rear-left leg:    x_RL(t) = A · sin(ωt + π)
Rear-right leg:   x_RR(t) = A · sin(ωt)
```

The phase shifts between legs determine the gait pattern (walk, trot, bound, gallop). Getting these shifts wrong causes the robot to stumble — the graphical left/right shift of Section 0.3 directly corresponds to leg timing offsets.

---

### 19.5 IMU Signal Processing — Trig Identities in Rotation

Inertial Measurement Units (IMUs) measure angular velocity (gyroscope) and linear acceleration (accelerometer). Converting these raw readings to orientation angles involves:

**Pythagorean identity:**
```
sin²θ + cos²θ = 1
```
This is used in complementary filters and Kalman filters to normalise rotation quaternions — ensuring the unit constraint is maintained numerically.

**Double angle formulas:**
```
sin(2θ) = 2 sin θ cos θ
cos(2θ) = cos²θ − sin²θ
```
These appear in rotation matrix products when composing successive rotations — for example, computing the orientation of a gimbal after two sequential rotations.

---

### 19.6 Fourier Analysis — Decomposing Any Signal into Trig Functions

Any periodic signal — a sensor reading, a motor current trace, a vibration spectrum — can be decomposed into a sum of sines and cosines (Fourier series):

```
f(t) = a₀ + Σ [aₙ cos(nωt) + bₙ sin(nωt)]
```

This is a direct application of every concept in Section 0.3:
- Each term is a trig function with its own amplitude (aₙ, bₙ) and period (2π/nω)
- Phase information is encoded in the ratio of sin/cos coefficients
- ASTC and reference angles are used when evaluating these at specific angles

**In robotics:** Fourier decomposition is used to identify vibration modes in robot structures, filter noise from encoder signals, and design resonance-avoiding controllers.

---

### 19.7 Vertical Shift (D) — Bias and Offset Calibration

The d parameter (vertical shift) has a direct physical analogue: **sensor bias** or **DC offset**.

An accelerometer at rest should read 0 m/s² on its horizontal axes, but due to manufacturing imperfection it reads 0.03 m/s². This is a vertical shift d = 0.03. Calibration removes it before the signal is used — exactly as we translate a function back to its natural midline.

Similarly, a motor may produce non-zero torque at zero command due to friction bias. The compensation term that removes it is a vertical shift in the force/torque mapping.

---

### 19.8 Co-terminal Angles in Encoder Wrapping

Rotary encoders (position sensors for motors) typically report angles modulo 2π or modulo 360°. When a joint crosses the 0/2π boundary, the encoder "wraps" — reporting, say, 0.01 rad after having been at 6.27 rad.

This is co-terminal angle arithmetic. The control software must detect and unwrap these jumps:

```
If  |θ_new − θ_old| > π:
    θ_corrected = θ_new + 2π   (or − 2π)
```

Understanding co-terminal angles is essential for implementing encoder unwrapping in firmware, preventing phantom velocity spikes in control loops.
