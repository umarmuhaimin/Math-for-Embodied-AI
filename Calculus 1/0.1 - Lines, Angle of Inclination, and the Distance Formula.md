## 1. Slope of a Line

### Concept

A line is defined by two properties: it is perfectly **straight** and it extends **infinitely**. To describe a line fully, you need either:
- **Two points** on the line, or
- **One point** and the **slope**

**Slope** describes how steeply a line rises or falls — it is the ratio of vertical change (rise) to horizontal change (run).

### Derivation

Given two points on a line:

```
P₁ = (x₁, y₁)      P₂ = (x₂, y₂)
```

```
        rise     Δy     y₂ - y₁
m  =  ——————  = ——— =  —————————
        run      Δx     x₂ - x₁
```

> **Why use subscripts instead of real numbers?** Because we want the formula to work for *any* two points — not just a specific pair. This is the generalisation principle that underlies all formula derivation.

### Worked Example

Find the slope through (-2, -3) and (8, 2):

```
       2 - (-3)      2 + 3      5        1
m  =  ——————————  = ———————  = ———  =  ———
       8 - (-2)      8 + 2      10       2
```

**m = 0.5**

> Note: It doesn't matter which point you assign as (x₁, y₁). Reversing the labels flips both numerator and denominator signs simultaneously — the result is the same slope.

### Sign Interpretation

| Slope (m) | Meaning |
|-----------|---------|
| m > 0 | Line rises left to right |
| m < 0 | Line falls left to right |
| m = 0 | Horizontal line |
| m undefined | Vertical line (division by zero) |

## 2. Equation of a Line

### Derivation of Point-Slope Form

Starting from the slope formula:

```
       y₂ - y₁
m  =  —————————
       x₂ - x₁
```

Fix one point as a specific known point (x₁, y₁) and let the second point float to any arbitrary point (x, y):

```
       y - y₁
m  =  —————————
       x - x₁
```

Multiply both sides by (x - x₁):

```
y - y₁  =  m(x - x₁)
```

This is **Point-Slope Form**.

### Point-Slope Form

```
y - y₁  =  m(x - x₁)
```

**Required inputs:** one known point (x₁, y₁) and slope m.

> This is the most direct and flexible equation form — use it whenever you know a point and a slope.

### Worked Example

Find the equation of the line through (-2, -3) and (8, 2):

Step 1 — Find the slope (from Section 1 above):  m = 1/2

Step 2 — Choose one point, e.g. (8, 2), and plug into point-slope:

```
y - 2  =  ½(x - 8)
y - 2  =  ½x - 4
    y  =  ½x - 2
```

## 3. Forms of a Line

### Slope-Intercept Form

Expanding and rearranging point-slope gives:

```
y  =  mx + b
```

| Symbol | Meaning |
|--------|---------|
| m | Slope |
| b | y-intercept (where the line crosses the y-axis, at x = 0) |

**Graphing from slope-intercept:**
1. Plot the y-intercept (0, b) on the y-axis
2. From that point, apply the slope: rise over run
3. Positive slope → go up; Negative slope → go down
4. Always move right (positive x-direction)

**Example:** y = ½x - 2

- y-intercept: (0, -2) → plot a point here
- Slope ½ → go up 1, right 2 → plot second point
- Draw the line through both points

### Standard Form

```
Ax + By + C = 0      or      Ax + By = C
```

**Converting to slope-intercept:** Solve for y.

**Example:** 4x + 2y - 3 = 0

```
2y  =  3 - 4x
 y  =  3/2 - 2x
 y  =  -2x + 3/2
```

So m = -2, b = 3/2.

**Cover-up method for intercepts (useful for graphing from standard form):**

Given 4x + 2y = 3:
- x-intercept: set y = 0 → 4x = 3 → x = 3/4
- y-intercept: set x = 0 → 2y = 3 → y = 3/2

Plot these two intercepts and draw the line — quicker than dealing with fractional slopes.

## 4. Special Lines: Horizontal and Vertical

### Horizontal Line

```
y = c     (c is a constant)
```

- Slope = 0
- Parallel to x-axis
- Crosses y-axis at (0, c)
- Cannot be vertical (a horizontal line cannot intercept another horizontal line)

**Diagram:**
```
   y
   |
c  |————————————————
   |
   +————————————————→ x
```

### Vertical Line

```
x = c     (c is a constant)
```

- Slope = undefined (division by zero)
- Parallel to y-axis
- Crosses x-axis at (c, 0)
- Cannot be horizontal

**Diagram:**
```
   y
   |   |
   |   |
   |   |
   +———+————————————→ x
       c
```

> **Memory rule:** The variable present tells you which axis the line is parallel to — and where it intercepts the *other* axis. `y = c` has only y → crosses y-axis. `x = c` has only x → crosses x-axis.

## 5. Parallel and Perpendicular Lines

### Parallel Lines

**Definition:** Lines that never intersect — they run in the same direction forever.

**Condition:** Equal slopes

```
l₁ ∥ l₂   ⟺   m₁ = m₂
```

The y-intercepts (and x-intercepts) will always differ — otherwise the lines would be the same line.

**Diagram:**
```
   y
   |  /  /
   | /  /      Two lines: same slope, different intercepts
   |/  /
   +———————→ x
```

### Perpendicular Lines

**Definition:** Lines that intersect at exactly 90°.

**Condition:** Slopes are negative reciprocals of each other

```
l₁ ⊥ l₂   ⟺   m₁ × m₂ = -1   ⟺   m₂ = -1/m₁
```

If one line has slope m, the perpendicular slope is -1/m.

**Examples:**
| Slope m₁ | Perpendicular m₂ |
|----------|-----------------|
| 2 | -1/2 |
| -2/3 | 3/2 |
| 1 | -1 |
| 1/4 | -4 |

**Diagram:**
```
   y
   | /
   |/
───+────→ x
   |\
   | \
```

### Worked Example: Parallel and Perpendicular

Find the line through (6, 7) parallel to 2x + 3y = 12, then also perpendicular.

**Step 1 — Find slope of the given line:**

```
3y  =  12 - 2x
 y  =  4 - (2/3)x
```

m₁ = -2/3

**Step 2 — Parallel line through (6, 7):**

Parallel → same slope: m₂ = -2/3

```
y - 7  =  -2/3(x - 6)
y - 7  =  -2/3 x + 4
    y  =  -2/3 x + 11     →  (l₂, parallel)
```

- y-intercept: 11
- x-intercept: set y = 0 → 2/3 x = 11 → x = 16.5

**Step 3 — Perpendicular line through (6, 7):**

Perpendicular → negative reciprocal: m₂ = 3/2

```
y - 7  =  3/2(x - 6)
y - 7  =  3/2 x - 9
    y  =  3/2 x - 2     →  (l₂, perpendicular)
```

**Finding the intersection point between parallel's reference line l₁ and l₂ (perpendicular):**

```
l₁: y = -2/3 x + 4
l₂: y =  3/2 x - 2

-2/3 x + 4  =  3/2 x - 2
         6  =  13/6 x
         x  =  36/13

         y  =  3/2 · (36/13) - 2  =  54/13 - 26/13  =  28/13
```

Intersection: (36/13, 28/13)

## 6. Angle of Inclination

### Concept

The **angle of inclination** (θ, "theta") is the angle a line makes with the positive x-axis, measured counter-clockwise.

```
   y
   | /
   |/ θ
───+————→ x
```

### Relationship to Slope

Using SOHCAHTOA on the right triangle formed by Δx (adjacent), Δy (opposite), and the line segment (hypotenuse):

```
tan θ  =  opposite / adjacent  =  Δy / Δx  =  rise / run  =  m
```

Therefore:

```
m  =  tan θ         (slope from angle)
θ  =  tan⁻¹(m)     (angle from slope)
```

### Direction: Slope → Angle

Given θ = 30° (or π/6):

```
m  =  tan(π/6)
   =  sin(π/6) / cos(π/6)
   =  (1/2) / (√3/2)
   =  1/√3
   =  √3/3
```

### Direction: Angle → Slope (Inverse)

Given m = -1, find θ:

```
tan θ  =  -1
```

Step 1 — Find the reference angle:

```
tan⁻¹(1)  =  π/4    (principle angle, ignoring sign)
```

Step 2 — Determine correct quadrant using CAST rule:

Tangent is negative in quadrants 2 and 4.

```
Quadrant 2:  θ  =  π - π/4  =  3π/4   (= 135°)
Quadrant 4:  θ  =  2π - π/4 =  7π/4   (= 315°)
```

Both are valid angles of inclination for slope m = -1. Either answer is acceptable — they correspond to the same line viewed from different directions.

### CAST Rule (Quadrant Sign Reference)

```
        S | A          Quadrant 2 | Quadrant 1
       ———+———         ——————————+——————————
        T | C          Quadrant 3 | Quadrant 4
```

| Quadrant | Positive functions |
|----------|--------------------|
| 1 (All) | sin, cos, tan all positive |
| 2 (Sin) | sin positive only |
| 3 (Tan) | tan positive only |
| 4 (Cos) | cos positive only |

### Quadrant Angle Formulas

| Quadrant | Angle formula (using reference φ) |
|----------|-----------------------------------|
| 1 | φ |
| 2 | π - φ |
| 3 | π + φ |
| 4 | 2π - φ |

> **Why does this matter for Calculus?** Slopes and tangent lines appear everywhere in differentiation. The angle of inclination connects the algebraic slope directly to geometric rotation — critical for understanding derivatives visually.

## 7. Distance Formula

### Derivation (Pythagorean Theorem)

Given two points P₁ = (x₁, y₁) and P₂ = (x₂, y₂):

Form a right triangle:
- Horizontal leg = x₂ - x₁ = Δx
- Vertical leg = y₂ - y₁ = Δy
- Hypotenuse = D (the distance we want)

By the Pythagorean theorem (a² + b² = c²):

```
D²  =  (x₂ - x₁)²  +  (y₂ - y₁)²
```

Taking the positive square root (distance cannot be negative):

```
D  =  √[(x₂ - x₁)²  +  (y₂ - y₁)²]
```

**Diagram:**
```
   y
   |              P₂(x₂,y₂)
   |             /|
   |            / |
   |       D   /  |  Δy = y₂-y₁
   |          /   |
   |     P₁  /    |
   |  (x₁,y₁)————+
   |         Δx = x₂-x₁
   +————————————————→ x
```

> **Critical note:** You cannot distribute the square root across addition. √(a² + b²) ≠ a + b. You must compute the full expression under the radical before taking the root.

### Worked Example

Distance between (-2, -3) and (8, 2):

```
D  =  √[(8-(-2))² + (2-(-3))²]
   =  √[(10)² + (5)²]
   =  √[100 + 25]
   =  √125
   =  5√5
   ≈  11.18
```

## 8. Key Formulas at a Glance

```
SLOPE
         y₂ - y₁
  m  =  —————————
         x₂ - x₁


POINT-SLOPE FORM
  y - y₁  =  m(x - x₁)


SLOPE-INTERCEPT FORM
  y  =  mx + b


ANGLE OF INCLINATION
  m  =  tan θ
  θ  =  tan⁻¹(m)


PARALLEL
  m₁  =  m₂


PERPENDICULAR
  m₁ × m₂  =  -1
  m₂  =  -1/m₁

  D  =  √[(x₂-x₁)² + (y₂-y₁)²]
```

## 9. Embodied AI Connections

The concepts in Section 0.1 are not just review algebra — they form the direct mathematical skeleton of systems that physically move through the world. Below are the key applications in Embodied AI and robotics engineering.

---

### 9.1 Coordinate Frames and Spatial Reasoning

Every robot operates in a coordinate frame. Whether a robotic arm in manufacturing or a mobile robot navigating a warehouse, the position of every joint, obstacle, and target is a point (x, y) or (x, y, z) in some frame. The slope between two such points is the fundamental building block of direction of travel, joint orientation, and geometric path planning.

**Directly used:** slope formula (Δy/Δx), distance formula, point representation

---

### 9.2 Velocity as Slope — The Pre-Calculus Foundation of Derivatives

In Calculus, the derivative of a position function at any instant gives instantaneous velocity. That derivative is *defined* as the slope of the tangent line to the position curve. Every time you compute velocity in a robotic system — a servo's angular velocity, a drone's groundspeed, a joint's rate of change — you are computing slope.

```
Conceptual chain:
  slope of secant (two points, Section 0.1)
  → slope of tangent (limit as Δx → 0, Calculus 1 core)
  → instantaneous velocity (robotics: joint/end-effector speed)
```

**Directly used:** slope formula — the seed of all differential calculus

---

### 9.3 Wheel Odometry and Dead Reckoning

Mobile robots estimate their position using wheel encoders (odometry). After each timestep, the robot's displacement is a vector — described by a change in x and y. The robot's heading is the angle that vector makes with a reference axis, which is the angle of inclination θ.

```
New heading:  θ  =  tan⁻¹(Δy / Δx)
```

This is exactly Section 0.1's angle of inclination formula, applied in real-time at every encoder tick. Dead reckoning chains many such steps together to estimate total displacement — critical when GPS is unavailable (indoors, tunnels, underwater).

**Directly used:** angle of inclination, slope-to-angle conversion

---

### 9.4 LiDAR Surface Normal Estimation

LiDAR sensors produce point clouds — thousands of (x, y) or (x, y, z) coordinate pairs per second. To determine the orientation of a surface (e.g., a wall, a ramp, a person's limb), the robot fits a line through nearby points and computes its slope. The **surface normal** — the direction perpendicular to that surface — is the perpendicular line from Section 0.1.

```
If fitted surface has slope m,
then surface normal has slope  -1/m   (perpendicular line formula)
```

This is used in:
- Grasp planning (approach angle for a robot hand)
- Terrain analysis (slope of ground for legged locomotion)
- 3D reconstruction (surface orientation for mesh generation)

**Directly used:** perpendicular lines, slope formula, distance formula (point fitting)

---

### 9.5 Inverse Kinematics (IK) — Geometric Approach

Inverse kinematics is the problem of computing joint angles that position an end-effector (robot hand, tool, sensor) at a desired location. In 2D, this involves computing the angle each link makes with the horizontal — which is the angle of inclination of the line segment representing each link.

```
Joint angle θ_i  =  tan⁻¹(Δy_i / Δx_i)
```

In 3D, this extends to atan2 (the four-quadrant version of arctan, using the CAST rule logic), and ultimately to rotation matrices and quaternions built on top of trigonometric foundations introduced here.

**Directly used:** angle of inclination formula, tan⁻¹, quadrant analysis

---

### 9.6 Camera Calibration and Computer Vision

In vision-based robotics, the camera captures images as a 2D grid of pixels. Recognising edges, estimating depth, and projecting 3D world coordinates to 2D image coordinates all rely on line equations. Edge detection algorithms (Canny, Hough Transform) work by finding lines in pixel space — their slope and intercept are computed directly using the formulas from this section.

**Hough Transform** explicitly parameterises lines in (m, b) or (ρ, θ) space — both derived directly from slope-intercept and angle-of-inclination formulas.

**Directly used:** slope-intercept form, angle of inclination

---

### 9.7 Path Planning — Straight-Line Trajectories

In motion planning, a common primitive is a straight-line path between two waypoints in configuration space or Cartesian space. Given start point A = (x₁, y₁) and goal B = (x₂, y₂):

```
Direction vector:   (Δx, Δy)  =  (x₂-x₁, y₂-y₁)
Path slope:         m  =  Δy / Δx
Path length:        D  =  √(Δx² + Δy²)      ← distance formula
Heading angle:      θ  =  tan⁻¹(Δy / Δx)
```

These four values define the straight-line primitive. More complex paths (splines, Bezier curves) are built by chaining and smoothing these primitives — every one of which starts from the Section 0.1 toolkit.

**Directly used:** slope, distance formula, angle of inclination, point-slope form

---

### 9.8 Sensor Fusion — Linear Models

Kalman Filters (the gold standard for sensor fusion in robotics) model state evolution as linear equations. In their simplest form, the prediction step propagates state along a *line* in state space — defined by a slope (the system matrix A) and intercept (the control input Bu). Developing intuition for linear relationships now pays off directly when studying state-space equations and Kalman filtering later.

**Directly used:** linear equation structure, slope-intercept concept extended to vectors and matrices
